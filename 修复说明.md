# 数据库配置问题修复说明

## 问题原因

从截图可以看到以下错误：
1. `DB_HOST`, `DB_NAME`, `DB_USER`, `DB_PASS` 常量未定义
2. 数据库连接失败

## 根本原因

1. **缺少 database.php 文件** - 只有 database.example.php 示例文件
2. **数据库配置常量未定义** - 代码中使用常量，但配置文件返回数组

## 修复内容

### 1. 创建了 database.php 配置文件

位置：`app/backend/config/database.php`

内容：
```php
<?php
// 定义数据库连接常量
define('DB_HOST', 'localhost');
define('DB_PORT', '3306');
define('DB_NAME', 'xiaoshuowang');
define('DB_USER', 'root');
define('DB_PASS', 'your_password_here');
define('DB_CHARSET', 'utf8mb4');
define('DB_PREFIX', '');

// 返回配置数组（兼容其他代码）
return [
    'host' => DB_HOST,
    'port' => DB_PORT,
    'database' => DB_NAME,
    'username' => DB_USER,
    'password' => DB_PASS,
    'charset' => DB_CHARSET,
    'prefix' => DB_PREFIX,
    'options' => [
        PDO::ATTR_ERRMODE => PDO::ERRMODE_EXCEPTION,
        PDO::ATTR_DEFAULT_FETCH_MODE => PDO::FETCH_ASSOC,
        PDO::ATTR_EMULATE_PREPARES => false,
    ],
];
```

### 2. 改进了错误处理

在 settings.php、ai_settings.php、backup.php 中：
- 检查配置文件是否存在
- 检查常量是否已定义
- 显示友好的错误页面

### 3. 创建了数据库表初始化脚本

位置：`app/admin/init_tables.php`

用于创建以下表：
- `system_settings` - 系统设置表
- `ai_settings` - AI设置表
- `sys_log` - 系统日志表
- `user_favorite` - 用户收藏表

## 部署步骤

### 第一步：修改数据库配置

编辑 `app/backend/config/database.php` 文件：

```php
define('DB_HOST', '您的数据库主机');      // 如: localhost
define('DB_NAME', '您的数据库名');        // 如: xiaoshuowang
define('DB_USER', '您的数据库用户名');    // 如: root
define('DB_PASS', '您的数据库密码');      // 如: 123456
```

### 第二步：初始化数据库表

访问：`http://您的域名/admin/init_tables.php`

这将自动创建所需的数据库表。

### 第三步：使用新功能

现在可以正常访问：
- 系统设置：`/admin/settings.php`
- AI设置：`/admin/ai_settings.php`
- 数据库备份：`/admin/backup.php`

## 注意事项

1. **必须先修改 database.php 中的数据库配置**
2. **必须运行 init_tables.php 初始化数据库表**
3. **确保数据库用户有创建表的权限**
